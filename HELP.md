# 課題リポジトリの使い方

## 0. GitHubからURLをコピーする方法

- GitHubでリポジトリのページを開く  
- 緑の「Code」ボタンをクリック  
- 「HTTPS」のURLをコピーする  

以下の手順に従って、課題のリポジトリをクローンし、CLionで開いて実装を進めてください。

## 1. リポジトリをクローンする方法

ターミナルまたはコマンドプロンプトで、以下のコマンドを実行してリポジトリをクローンします。

```bash
git init
git clone <リポジトリのURL>
```

まず `git init` でリポジトリを初期化し、その後に `git clone` を実行してください。  
`<リポジトリのURL>`は配布されたリポジトリのURLに置き換えてください。

## 2. CLionでプロジェクトを開く方法

1. CLionを起動します。
2. メニューから「Open」を選択し、クローンしたリポジトリのフォルダを選択します。
3. プロジェクトが読み込まれ、CMakeの設定が自動的に行われます。

## 3. `main.cpp` を実行してテストする手順

1. CLionの右上の実行構成から`main`を選択します。
2. 「Run」ボタンを押してプログラムを実行します。
3. 出力や動作を確認して、正しく動作するかテストします。

## 4. Gitの基本コマンド表

| コマンド | 説明 |
|----------|------|
| `git clone <URL>` | リポジトリをコピーして取得 |
| `git status` | 現在の状態を確認（変更点やステージ状況） |
| `git add .` | すべての変更をステージに追加 |
| `git commit -m "メッセージ"` | ステージされた変更をコミット |
| `git pull` | リモートリポジトリから最新を取得してマージ |
| `git push` | リモートリポジトリに送信 |

## 5. GitHubへのプッシュ手順

1. GitHubで新しいリポジトリを作成する  
2. 作成後に表示されるコマンドをコピーする (`git remote add origin ...` など)  
3. ローカルのプロジェクトでそのコマンドを実行する  
4. 最後に `git push -u origin main` を実行する  

## 6. ローカルでのコミット例

作業内容をGitで管理し、リモートリポジトリに反映させます。

```bash
git add .
git commit -m "課題の実装"
git push
```

- `git add .` で変更をステージングします。
- `git commit -m "課題の実装"` でコミットメッセージを付けて保存します。
- `git push` でリモートリポジトリに変更を送信します。
